## Variabili d'ambiente da aggiungere in `.env.local`

```
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...
GOOGLE_REDIRECT_URI=https://<tuo-dominio>/api/google-calendar/callback
```

**Come ottenerle:**
1. Vai su [Google Cloud Console](https://console.cloud.google.com) в†’ API & Services в†’ Credentials
2. Crea un OAuth 2.0 Client ID di tipo "Web application"
3. Aggiungi `GOOGLE_REDIRECT_URI` come "Authorized redirect URI"
4. Copia Client ID e Client Secret in `.env.local`
5. Abilita l'API Google Calendar nel progetto Cloud Console

## Migrazioni SQL da eseguire nel dashboard Supabase

Esegui il contenuto di `supabase/migrations/20260514_integrazioni.sql`:

```sql
CREATE TABLE IF NOT EXISTS integrazioni (
  id           uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  autista_id   uuid NOT NULL REFERENCES autisti(id) ON DELETE CASCADE,
  provider     text NOT NULL,
  access_token text,
  refresh_token text,
  expires_at   timestamptz,
  calendar_id  text,
  created_at   timestamptz DEFAULT now(),
  UNIQUE(autista_id, provider)
);

ALTER TABLE integrazioni ENABLE ROW LEVEL SECURITY;

CREATE POLICY "integrazioni_own" ON integrazioni
  FOR ALL USING (autista_id = auth.uid())
  WITH CHECK (autista_id = auth.uid());
```

Verifica con:
```sql
SELECT tablename FROM information_schema.tables WHERE table_name = 'integrazioni';
```

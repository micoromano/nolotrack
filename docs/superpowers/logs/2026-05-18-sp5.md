## Migrazione SQL da eseguire nel Supabase dashboard

File: `supabase/migrations/20260518_sp5_ruoli.sql`

Copia e incolla nel **SQL Editor** di Supabase, poi esegui. La migrazione:
1. Crea le tabelle `ruoli` e `ruolo_permessi`
2. Aggiunge la colonna `ruolo_id` alla tabella `autisti`
3. Inserisce i 3 ruoli di default con i rispettivi permessi per le 11 sezioni
4. Abilita RLS su `ruoli` e `ruolo_permessi` con policy di lettura per utenti autenticati

**Verifica dopo l'esecuzione:**
```sql
SELECT r.nome, rp.sezione, rp.can_view, rp.can_edit
FROM ruoli r JOIN ruolo_permessi rp ON r.id = rp.ruolo_id
ORDER BY r.nome, rp.sezione;
-- Devono apparire 33 righe
```

## Assegnazione ruolo admin a Marco

Dopo aver eseguito la migrazione, esegui nel SQL Editor:
```sql
UPDATE autisti
SET ruolo_id = (SELECT id FROM ruoli WHERE nome = 'admin')
WHERE email = 'marco.camelin@gmail.com';
```

Senza questa istruzione, Marco vedrà la navbar con i permessi di default "autista" e non vedrà la voce "Admin".

## RLS policies abilitate dalla migrazione

- `ruoli`: policy `ruoli_read` — SELECT per tutti gli utenti autenticati
- `ruolo_permessi`: policy `ruoli_permessi_read` — SELECT per tutti gli utenti autenticati

Le tabelle `autisti`, `corse`, `turni` ecc. mantengono le policy esistenti invariate.

## Note e decisioni implementative

- **Invite API fix**: Il piano originale usava `user.id` (l'admin) come ID placeholder nel record `autisti` dell'invitato. Corretto: si usa `invData.user.id` restituito da `inviteUserByEmail`, che è l'UUID reale del nuovo utente già creato da Supabase.
- **Mobile tabs**: Filtrati per permessi come il desktop. Usano `style.gridTemplateColumns` per il numero dinamico di colonne (Tailwind non può purgare classi dinamiche).
- **Task non implementato**: Il piano include "campo autista per dispatcher" in `corse/nuova/page.tsx` ma non fornisce codice specifico. Rinviato a iterazione successiva.
- **TypeScript check**: `npx tsc --noEmit` mostra errori pre-esistenti (missing `@types/node`, Next.js JSX setup) presenti anche nei file non modificati. Non introdotti da SP5.

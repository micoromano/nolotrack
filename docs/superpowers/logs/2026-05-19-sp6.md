## Raccomandazioni residue

1. **RLS autisti**: eseguire manualmente il SQL in `supabase/rls-check.sql` nel dashboard Supabase per verificare e applicare le policy mancanti.
2. **Rate limiting produzione**: il rate limiter in-memory si resetta ad ogni deploy. Per produzione, considerare Redis o Upstash per persistenza tra istanze.
3. **Playwright E2E**: configurare `TEST_EMAIL` e `TEST_PASSWORD` nell'ambiente CI (GitHub Actions secrets) per eseguire i test E2E in pipeline.
4. **Locale Intl**: la suite unit usa normalizzazione per tollerare differenze ICU tra Node.js/jsdom. In produzione il browser nativo gestisce correttamente la locale `it-IT`.

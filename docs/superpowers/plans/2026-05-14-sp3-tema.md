# SP3 — Tema Chiaro/Scuro

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Aggiungere toggle chiaro/scuro/sistema all'app, con look da desktop application (toolbar con breadcrumb, status bar, righe selezionabili).

**Architecture:** `next-themes` gestisce la classe `.dark`/`.light` su `<html>` e persiste in localStorage → `globals.css` aggiunge variabili `.light` → `layout.tsx` wrappa con ThemeProvider → Navbar riceve toggle select → aggiornamento visivo trasversale a tutte le pagine.

**Tech Stack:** `next-themes`, Tailwind v4 CSS variables, Next.js App Router

## Task 2: ThemeProvider

**Files:**
- Create: `components/theme-provider.tsx`

- [ ] **Step 1: Crea il componente**

```tsx
"use client";

import { ThemeProvider as NextThemesProvider } from "next-themes";

export function ThemeProvider({ children }: { children: React.ReactNode }) {
  return (
    <NextThemesProvider
      attribute="class"
      defaultTheme="dark"
      enableSystem
      themes={["dark", "light", "system"]}
    >
      {children}
    </NextThemesProvider>
  );
}
```

## Task 3: Aggiorna layout.tsx

**Files:**
- Modify: `app/layout.tsx`

- [ ] **Step 1: Rimuovi `dark` hardcoded e wrappa con ThemeProvider**

Sostituisci l'intero file:

```tsx
import type { Metadata } from "next";
import { IBM_Plex_Sans, IBM_Plex_Mono, DM_Serif_Display } from "next/font/google";
import { ThemeProvider } from "@/components/theme-provider";
import "./globals.css";

const ibmSans = IBM_Plex_Sans({
  variable: "--font-ibm-sans",
  weight: ["300", "400", "500", "600"],
  subsets: ["latin"],
});

const ibmMono = IBM_Plex_Mono({
  variable: "--font-ibm-mono",
  weight: ["400", "500"],
  subsets: ["latin"],
});

const dmSerif = DM_Serif_Display({
  variable: "--font-dm-serif",
  weight: "400",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "NoloTrack",
  description: "Gestione NCC — Noleggio Con Conducente",
};

export default function RootLayout({
  children,
}: Readonly<{ children: React.ReactNode }>) {
  return (
    <html
      lang="it"
      className={`${ibmSans.variable} ${ibmMono.variable} ${dmSerif.variable} h-full antialiased`}
      suppressHydrationWarning
    >
      <body className="min-h-full flex flex-col">
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  );
}
```

Nota: `suppressHydrationWarning` è necessario perché `next-themes` modifica la classe su `<html>` lato client dopo l'idratazione.

## Task 5: Toggle tema nella Navbar

**Files:**
- Modify: `components/navbar.tsx`

- [ ] **Step 1: Aggiungi hook useTheme e componente ThemeToggle**

In `components/navbar.tsx`, aggiungi l'import:

```tsx
import { useTheme } from "next-themes";
```

All'interno del componente `NavBar`, aggiungi:

```tsx
const { theme, setTheme } = useTheme();
```

- [ ] **Step 2: Aggiungi il select del tema nella sidebar desktop**

Nella sidebar desktop, subito prima del div con email utente e bottone disconnetti, aggiungi:

```tsx
<div className="px-3 pb-2">
  <select
    value={theme ?? "dark"}
    onChange={(e) => setTheme(e.target.value)}
    className="w-full bg-sidebar-accent border border-sidebar-border text-xs text-muted-foreground px-2 py-1.5 rounded focus:outline-none focus:ring-1 focus:ring-primary"
  >
    <option value="dark">🌙 Scuro</option>
    <option value="light">☀️ Chiaro</option>
    <option value="system">💻 Sistema</option>
  </select>
</div>
```

- [ ] **Step 3: Aggiungi toggle nel menu mobile**
